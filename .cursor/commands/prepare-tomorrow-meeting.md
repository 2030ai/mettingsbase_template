# Команда: Подготовить повестку встречи на завтра

## Описание
Автоматически готовит повестку встречи на завтра на основе результатов последней встречи из базы знаний.

## Алгоритм выполнения

1. **Определить дату завтра**
   - Выполнить команду `date +%Y.%m.%d` для получения текущей даты
   - Рассчитать дату завтра в формате `YYYY.MM.DD`

2. **Найти последнюю встречу**
   - Искать файлы встреч в `база знаний/встречи/` с датами до завтрашней
   - Найти самую последнюю встречу (по дате в имени файла)

3. **Прочитать результаты прошлой встречи**
   - Прочитать файл последней встречи
   - Извлечь ключевую информацию:
     * Решения (Decision Log)
     * Задачи (Action Items) с дедлайнами
     * Открытые вопросы (Open Questions)
     * Участники
     * Тема встречи
     * Следующие шаги

4. **Создать папку для завтрашней встречи**
   - Создать папку `база знаний/встречи/YYYY.MM/` если её нет
   - Формат: `база знаний/встречи/2026.01/`

5. **Сгенерировать повестку**
   - Использовать шаблон из `01_before/agenda_template.md`
   - Заполнить на основе данных из прошлой встречи:
     * **Метаданные**: дата завтра, участники из прошлой встречи
     * **Цель встречи**: связать с проверкой статуса задач и подготовкой к следующим шагам
     * **Ожидаемые outputs**: проверить готовность задач из прошлой встречи
     * **Вопросы для решения**: подтверждение готовности компонентов, закрытие блокеров
     * **Контекст (pre-reads)**:
       - Ссылка на протокол предыдущей встречи
       - Статус задач из прошлой встречи (особенно те, у которых дедлайн сегодня или завтра)
       - Открытые вопросы из прошлой встречи
     * **Тайминг**: адаптировать под проверку статуса и подготовку к следующим шагам

6. **Именование файла**
   - Формат: `база знаний/встречи/YYYY.MM/YYYY.MM.DD [тема встречи].md`
   - Тема: на основе цели встречи или продолжения предыдущей темы

## Правила заполнения повестки

### Цель встречи
- Если есть задачи с дедлайном сегодня/завтра → включить проверку готовности
- Если есть открытые вопросы → включить их закрытие
- Если есть следующий шаг из прошлой встречи → сделать его целью

### Контекст (pre-reads)
- **Что прочитать заранее:**
  - Протокол предыдущей встречи: `[путь к файлу прошлой встречи]`
  - Статус задач из прошлой встречи:
    * [Владелец]: [задача] (дедлайн [дата]) ← проверить готовность
  - Открытые вопросы:
    * [Вопрос] ([Владелец], срок до [дата])

- **Ограничения/условия:**
  - Дедлайны задач
  - Следующие важные даты/события
  - Любые ограничения из прошлой встречи

### Вопросы для решения
- Утвердить готовность компонентов к следующему этапу
- Выбрать формат/подход если есть альтернативы
- Согласовать детали процесса если они не были закрыты

### Тайминг
- 00:00–00:02 — Старт: цель, outputs, роли, правила
- 00:02–00:05 — Статус задач из прошлой встречи (проверка готовности компонентов)
- 00:05–00:15 — Решение 1: подтверждение готовности / выбор подхода
- 00:15–00:22 — Решение 2: финальный чек-лист / закрытие открытых вопросов
- 00:22–00:25 — Открытые вопросы: закрыть блокеры
- 00:25–00:30 — Итог: решения + задачи (кто-что-когда) + следующий шаг

## Пример использования

Пользователь говорит: "/prepare-tomorrow-meeting" или "подготовь повестку на завтра"

AI автоматически:
1. Определяет дату завтра
2. Находит последнюю встречу
3. Извлекает результаты
4. Создает папку и файл повестки
5. Заполняет повестку на основе шаблона и данных прошлой встречи
